# Almaty Clinic — психологический скрининг студентов

Двухэтапное тестирование: **HADS** (тревога + депрессия) → при ≥ 9 баллов по любой
шкале автоматически подключается **шкала депрессии Бека**. Все результаты
сохраняются в облачную базу и выгружаются в Excel из кабинета психолога.

## Возможности

- Вход по QR-коду, языки **РУС / ҚАЗ**
- Идентификация: ВУЗ, ФИО, ИИН (проверка 12 цифр)
- Два согласия (обработка данных + тестирование)
- HADS: раздельный автоподсчёт по тревоге и депрессии
- Автопереход на шкалу Бека по порогу ≥ 9
- Кабинет психолога: таблица прохождений, статистика, отметка риска
- Выгрузка всех записей в один файл **.xlsx**
- Централизованное облачное хранение (данные собираются со всех устройств)

## Архитектура

Фронтенд — статический файл `public/index.html`. Бэкенд — Express (`server.js`),
на Vercel запускается как serverless-функция (`api/index.js`). База — **Turso**
(облачный SQLite по протоколу libSQL).

> Почему не локальный файл SQLite: на Vercel файловая система только для чтения и
> обнуляется между запросами, поэтому файл `screening.db` там жить не может. Turso
> хранит данные в облаке и работает из serverless без нативной сборки.

## Деплой на Vercel (рекомендуется)

### 1. Создать базу Turso (бесплатно)

Через сайт **turso.tech**: зарегистрируйтесь → Create Database → скопируйте
**Database URL** (вида `libsql://...turso.io`) и создайте **Token**.

Либо через CLI:

```bash
curl -sSfL https://tur.so/install.sh | bash   # установка CLI
turso auth signup
turso db create almaty-clinic
turso db show almaty-clinic --url              # → TURSO_DATABASE_URL
turso db tokens create almaty-clinic           # → TURSO_AUTH_TOKEN
```

### 2. Залить проект на GitHub и подключить к Vercel

Импортируйте репозиторий в Vercel (New Project → Import). Сборка настраивается
автоматически по `vercel.json`.

### 3. Задать переменные окружения в Vercel

Project → Settings → Environment Variables:

| Переменная | Значение |
|---|---|
| `TURSO_DATABASE_URL` | из шага 1 |
| `TURSO_AUTH_TOKEN` | из шага 1 |
| `ADMIN_PASSWORD` | пароль входа в кабинет психолога |
| `ADMIN_TOKEN` | длинная случайная строка (необязательно) |

Нажмите **Redeploy**. Готово: таблица в базе создастся автоматически при первом
обращении, приложение открывается по адресу проекта, QR-код — в кабинете психолога.

## Локальный запуск

Требуется Node.js 18+.

```bash
npm install
cp .env.example .env       # можно не заполнять Turso — будет локальный файл screening.db
npm start                  # http://localhost:3000
```

Кабинет психолога: кнопка на стартовом экране, пароль из `ADMIN_PASSWORD`.

## API

| Метод | Путь | Доступ | Назначение |
|------|------|--------|------------|
| POST | `/api/login` | — | Вход психолога, возвращает токен |
| POST | `/api/records` | — | Приём результата теста (студент) |
| GET | `/api/records` | Bearer | Список всех записей |
| GET | `/api/records/stats` | Bearer | Счётчики |
| GET | `/api/export.xlsx` | Bearer | Файл Excel |
| DELETE | `/api/records` | Bearer | Очистка (осторожно) |

## Структура

```
api/index.js         — точка входа Vercel (экспорт Express-приложения)
server.js            — маршруты API + Excel
db.js                — подключение к Turso / локальному SQLite + схема
public/index.html    — приложение (тест, кабинет, QR)
vercel.json          — маршрутизация (/api → функция, статика → public)
package.json         — зависимости
.env.example         — настройки (пароли, Turso, порт)
.gitignore / LICENSE / .nvmrc / .editorconfig
```

## Замечания по данным

- Приложение — инструмент **скрининга**, не диагноз. Итог интерпретирует специалист.
- Хранятся персональные данные (ФИО, ИИН) — используйте HTTPS (на Vercel по умолчанию),
  ограничьте доступ к кабинету, соблюдайте законодательство РК о персональных данных.
- Пункт 9 шкалы Бека (мысли о самоповреждении) помечается отдельным флагом риска.
