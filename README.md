# Almaty Clinic — психологический скрининг студентов

Двухэтапное тестирование: **HADS** (тревога + депрессия) → при ≥ 9 баллов по любой
шкале автоматически подключается **шкала депрессии Бека**. Все результаты
сохраняются в базу данных и выгружаются в Excel из кабинета психолога.

## Возможности

- Вход по QR-коду, языки **РУС / ҚАЗ**
- Идентификация: ВУЗ, ФИО, ИИН (проверка 12 цифр)
- Два согласия (обработка данных + тестирование)
- HADS: раздельный автоподсчёт по тревоге и депрессии
- Автопереход на шкалу Бека по порогу ≥ 9
- Кабинет психолога: таблица всех прохождений, статистика, отметка риска
- Выгрузка всех записей в один файл **.xlsx**
- Централизованное хранение в базе (SQLite) — данные собираются со всех устройств

## Запуск локально

Требуется Node.js 18+.

```bash
npm install
cp .env.example .env      # откройте .env и задайте ADMIN_PASSWORD
npm start
```

Откройте http://localhost:3000

Кабинет психолога: на стартовом экране — «Кабинет психолога», пароль из `.env`
(`ADMIN_PASSWORD`).

## Развёртывание в интернете

1. Разместите проект на сервере (VPS, Render, Railway, Timeweb, любой Node-хостинг).
2. Задайте переменные окружения `ADMIN_PASSWORD` и `ADMIN_TOKEN` (длинный секрет).
3. Обязательно включите **HTTPS** (домен + сертификат) — передаются перс. данные.
4. Настройте регулярный бэкап файла `screening.db`.
5. QR-код для студентов возьмите в кабинете психолога — он ведёт на адрес сайта.

## API

| Метод | Путь | Доступ | Назначение |
|------|------|--------|------------|
| POST | `/api/login` | — | Вход психолога, возвращает токен |
| POST | `/api/records` | — | Приём результата теста (студент) |
| GET | `/api/records` | Bearer | Список всех записей |
| GET | `/api/records/stats` | Bearer | Счётчики |
| GET | `/api/export.xlsx` | Bearer | Файл Excel |
| DELETE | `/api/records` | Bearer | Очистка (осторожно) |

## Структура

```
server.js            — API + база + Excel + раздача фронтенда
public/index.html    — приложение (тест, кабинет, QR)
package.json         — зависимости
.env.example         — настройки (пароли, порт, путь к базе)
.gitignore           — исключения (node_modules, .env, *.db …)
LICENSE              — условия использования (шаблон)
screening.db         — создаётся автоматически при первом запуске (в git не хранится)
```

## Работа с Git

```bash
# репозиторий уже инициализирован, с первым коммитом
git remote add origin <URL-вашего-репозитория>
git branch -M main
git push -u origin main
```

Важно: файл `.env` с паролями в репозиторий **не коммитится** (см. `.gitignore`).
В репозитории хранится только `.env.example`. База `screening.db` также игнорируется —
на сервере она создаётся при первом запуске.

## Замечания по данным

- Приложение — инструмент **скрининга**, не диагноз. Итог интерпретирует специалист.
- База содержит перс. данные (ФИО, ИИН) — обеспечьте защиту доступа, HTTPS,
  бэкапы и соблюдение законодательства РК о персональных данных.
- Пункт 9 шкалы Бека (мысли о самоповреждении) помечается отдельным флагом риска
  для приоритетного внимания психолога.
